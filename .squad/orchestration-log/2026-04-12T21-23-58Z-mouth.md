## Deliverables
### 1. ASP.NET Core Web Project
- **Path:** `src/Waymarked.Web/`
- **Files:**
  - `Waymarked.Web.csproj` — Project file with YARP 2.2.0 dependency
  - `Program.cs` — YARP reverse proxy configuration + static file serving
  - `appsettings.json` — Routing rules for `/api/{**catch-all}` → `waymarked-api`
  - `wwwroot/index.html` — Single-page Leaflet application

### 2. Frontend UI/UX
- **Map Rendering:** Leaflet 1.9.4 (CDN) with OpenTopoMap tiles
- **Default View:** UK (54.0, -2.0, zoom 6)
- **Input Form:** Lat/lon text boxes for start/end points (no geocoding yet)
- **Route Display:** Blue polyline with auto-fit bounds
- **Stats Panel:** Distance (km/mi), time estimate, instruction count

### 3. YARP Reverse Proxy
- **Purpose:** Avoid CORS issues by proxying API calls through same origin
- **Route Rule:** `/api/{**catch-all}` → Aspire service `waymarked-api`
- **Discovery:** Service reference via Aspire, `WaitFor(api)` dependency

### 4. Aspire Integration
- **AppHost Setup:** Web project registered with `.WithReference(api).WaitFor(api)`
- **Project Reference:** Added ProjectReference in AppHost.csproj
- **Observability:** Called `AddServiceDefaults()` for health checks + telemetry

## Technical Decisions
1. **No Build Step:** Plain HTML/CSS/JS for rapid iteration (no npm/webpack)
2. **CDN Dependencies:** Leaflet + OpenTopoMap for zero-config map
3. **YARP for Proxy:** Simpler than CORS headers, keeps solution in C#
4. **Optional End Point:** Defaults to start lat/lon for round-trip routes
